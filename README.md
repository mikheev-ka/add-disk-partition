# Работа с дисками в Linux: Создание раздела и монтирование

Этот конспект содержит базовые шаги по добавлению нового диска, созданию на нем раздела, форматированию и настройке автоматического монтирования в системе Linux. Подходит для быстрого повторения или использования в качестве справочника.

---

## 📋 Основные шаги

### 1. Создание раздела
Допустим, в системе появился новый диск `/dev/sdb`. Необходимо создать на нем один раздел.

```bash
sudo fdisk /dev/sdb
Действия внутри fdisk:

Шаг	Действие	Описание
1	n	Создать новый раздел
2	p	Выбрать тип раздела: primary (основной)
3	1	Указать номер раздела
4	Enter	Использовать всё доступное пространство (принять значение по умолчанию)
5	w	Записать изменения и выйти
2. Форматирование раздела
Созданный раздел /dev/sdb1 необходимо отформатировать в файловую систему (например, ext4):

bash
sudo mkfs.ext4 /dev/sdb1
3. Монтирование раздела
Создайте точку монтирования и временно примонтируйте раздел:

bash
sudo mkdir -p /mnt/data
sudo mount /dev/sdb1 /mnt/data
4. Автоматическое монтирование при загрузке
Чтобы раздел монтировался автоматически после перезагрузки, добавьте запись в файл /etc/fstab.

Шаг 1: Узнайте UUID раздела (рекомендуется вместо имени устройства):

bash
sudo blkid /dev/sdb1
Шаг 2: Отредактируйте файл /etc/fstab:

bash
sudo nano /etc/fstab
Шаг 3: Добавьте одну из строк (используя полученный UUID или имя устройства):

bash
# Рекомендуемый способ (с UUID)
UUID=ваш-uuid-здесь /mnt/data ext4 defaults 0 2

# Альтернативный способ (с именем устройства)
/dev/sdb1 /mnt/data ext4 defaults 0 2
Шаг 4: Проверьте, что запись корректна:

bash
sudo mount -a
Если ошибок нет, раздел будет автоматически монтироваться при старте системы.

⚠️ Важное замечание
Данный подход создает раздел фиксированного размера, равного размеру диска.

Если в будущем добавится еще один диск, вы не сможете просто расширить существующую точку монтирования /mnt/data без переноса данных или использования более сложных решений:

LVM (Logical Volume Manager)

RAID (массивы дисков)

ZFS или Btrfs

Новый диск придется монтировать отдельно в другую директорию.

📊 Полезные команды
Команда	Описание
lsblk	Показать все блочные устройства и точки монтирования
sudo fdisk -l	Вывести таблицы разделов на всех дисках
df -h	Показать использование дискового пространства смонтированных файловых систем
sudo blkid	Показать UUID и типы файловых систем для всех разделов
