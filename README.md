# Работа с дисками в Linux: Создание раздела и монтирование

Этот конспект содержит базовые шаги по добавлению нового диска, созданию на нем раздела, форматированию и настройке автоматического монтирования в системе Linux. Подходит для быстрого повторения или использования в качестве справочника.

## Основные шаги

### 1. Создание раздела
Допустим, в системе появился новый диск `/dev/sdb`. Необходимо создать на нем один раздел.

```bash
sudo fdisk /dev/sdb
Действия внутри fdisk:

Нажмите n — создать новый раздел.

Выберите тип раздела: primary (основной).

Укажите номер раздела: 1.

Используйте всё доступное пространство (нажмите Enter, чтобы принять значение по умолчанию).

Нажмите w — записать изменения и выйти.

2. Форматирование раздела
Созданный раздел /dev/sdb1 необходимо отформатировать в файловую систему (например, ext4):

bash
sudo mkfs.ext4 /dev/sdb1
3. Монтирование раздела
Создайте точку монтирования и временно примонтируйте раздел:

bash
sudo mkdir -p /mnt/data
sudo mount /dev/sdb1 /mnt/data
4. Автоматическое монтирование при загрузке
Чтобы раздел монтировался автоматически после перезагрузки, добавьте запись в файл /etc/fstab.

Сначала узнайте UUID раздела (рекомендуется вместо имени устройства):

bash
sudo blkid /dev/sdb1
Затем отредактируйте файл:

bash
sudo nano /etc/fstab
Добавьте строку (используя полученный UUID или имя устройства):

text
UUID=ваш-uuid-здесь /mnt/data ext4 defaults 0 2
Или, если используете имя устройства:

text
/dev/sdb1 /mnt/data ext4 defaults 0 2
Проверьте, что запись корректна:

bash
sudo mount -a
Если ошибок нет, раздел будет автоматически монтироваться при старте системы.

Важное замечание
Данный подход создает раздел фиксированного размера, равного размеру диска.
Если в будущем добавится еще один диск, вы не сможете просто расширить существующую точку монтирования /mnt/data без переноса данных или использования более сложных решений (например, LVM, RAID, ZFS).
Новый диск придется монтировать отдельно.
